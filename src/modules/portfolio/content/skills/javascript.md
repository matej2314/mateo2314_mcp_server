Logikę po stronie przeglądarki i w Node.js realizuję w JavaScript — **język**, na którym opiera się cały mój frontend (React, Next.js) oraz część backendu. Potrafię **pisać modułowy kod** (ESM, importy), pracować z **API przeglądarki i DOM** tam, gdzie framework nie wystarcza, oraz projektować **przepływ asynchroniczny** (`async`/`await`, Promises, `fetch`) bez „callback hell”.

Znam **semantykę języka** (zakres, domknięcia, `this`, prototypy) w zakresie potrzebnym w codziennej pracy oraz stosuję **praktyki utrzymaniowe**: czytelne nazewnictwo, unikanie mutacji tam, gdzie szkodzą, świadome użycie `Array`/`Object` helpers i narzędzi devtools — oraz świadome ograniczenia: dynamiczny JS nie zastępuje walidacji po stronie serwera ani testów przy krytycznej logice.

Doświadczenie z:
- aplikacjami SPA i SSR w ekosystemie React / Next.js
- skryptami Node.js (narzędzia, proste API, integracje)
- współpracą z TypeScript i stopniową migracją z „czystego” JS
- debugowaniem i profilowaniem w przeglądarce przy problemach wydajnościowych
