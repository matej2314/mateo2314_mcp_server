Wygląd i układ stron buduję w CSS — **warstwa prezentacji**, w której łączę typografię, kolory, odstępy i responsywność bez mieszania logiki biznesowej z markupem. Potrafię **układać interfejsy** (Flexbox, Grid, pozycjonowanie), pisać **media queries**,stosuję podejście mobile-first oraz dbam o **stany interakcji** (hover, focus, disabled) pod kątem UX i dostępności.

Znam **kaskadę i specyficzność** na tyle, by unikać „wojny selektorów”, oraz stosuję **praktyki utrzymaniowe**: zmienne CSS (`custom properties`) i tokeny wizualne, sensowne BEM lub inne konwencje nazewnicze tam, gdzie arkusze rosną, reset/normalize świadomie dobrane do projektu oraz świadome ograniczenia — czysty CSS ma swoje granice; przy dużych systemach UI warto łączyć go z komponentami (React) lub frameworkami utility (np. Tailwind) zamiast puchnąć jednym globalnym plikiem.

Doświadczenie z:
- layoutami responsywnymi i komponentami UI w projektach webowych
- integracją z React (CSS Modules, styled-components, Tailwind) i design tokenami
- dostępnością (kontrast, focus visible, reduced motion tam, gdzie ma sens)
- debugowaniem w DevTools (computed styles, nakładające się reguły)
