## Opis kursu

Przeszedłem [Claude Code in Action](https://anthropic.skilljar.com/claude-code-in-action) — praktyczny kurs od Anthropic, który pokazuje, jak realnie wykorzystać Claude Code w codziennej pracy programistycznej. To nie jest teoretyczne gadanie o AI, tylko konkretne techniki i integracje: od podstawowej architektury asystentów kodujących, przez zarządzanie kontekstem, aż po budowanie własnych automatyzacji i podpinanie zewnętrznych narzędzi przez MCP servery.

Kurs prowadzi przez całą strukturę: jak Claude Code operuje na narzędziach (tools), jak utrzymywać kontekst rozmowy przy złożonych zadaniach, jak komunikować zmiany wizualnie i — co najważniejsze — jak rozszerzyć funkcjonalność o custom commands, hooks i integrację z GitHub. Dużo praktyki: setup projektu, dodawanie kontekstu, wprowadzanie zmian, a wszystko w realnych scenariuszach developmentu.

## Co wynoszę po kursie

- **Zrozumienie architektury** — wiem, jak AI asystenci wchodzą w interakcję z kodem przez system narzędzi i jakie są fundamenty techniczne tej współpracy.
- **Opanowanie tool use** — umiem łączyć kilka narzędzi w jednym flow, żeby obsłużyć wieloetapowe zadania programistyczne zamiast walczyć z pojedynczymi promptami.
- **Strategie zarządzania kontekstem** — nauczyłem się, jak skutecznie referencjonować zasoby projektu i utrzymywać istotny kontekst w długich sesjach kodowania.
- **Workflow z wizualizacjami** — wiem, jak używać obrazów do komunikowania zmian w interfejsie i kiedy włączać zaawansowane tryby planowania przy większych refaktorach.
- **Własne automatyzacje** — potrafię tworzyć custom commands i hooki, które odblokowują powtarzalne zadania i dopasowują Claude Code do mojego flow.
- **Rozszerzenia przez MCP** — umiem podpinać zewnętrzne serwisy (np. automatyzacja przeglądarki) przez MCP servers, co daje nowe możliwości poza standardowym zestawem narzędzi.
- **Integracja z GitHub** — mam setup do automatycznego code review i AI assistance wplecionego w procesy version control — to zmienia sposób pracy w zespole.
- **Tryby myślenia i planowania** — rozpoznaję, kiedy użyć różnych poziomów rozumowania w zależności od złożoności problemu (thinking vs planning modes).

Kurs dał mi konkretny toolkit do pracy z AI w kodzie — teraz mniej eksperymentuję, a więcej świadomie wykorzystuję możliwości Claude Code w produkcyjnych projektach.
